# IconBuilds Deployment

IconBuilds is a static HTML/CSS/JS frontend with a Vercel-compatible Node API at `api/index.js`.

## 1. Domain

Production domain:

```txt
https://minestore.org
```

Update `config.js` if the domain changes:

```js
site: {
  domain: "minestore.org",
  url: "https://minestore.org"
}
```

## 2. Required Vercel Environment Variables

Set these in Vercel Project Settings -> Environment Variables.

```txt
SESSION_SECRET=use-a-long-random-secret
ADMIN_EMAILS=thestickboy@example.com,itzkuroyt@example.com
```

Admin access is checked server-side by verified account email and role. Replace the example emails with the real Google/email accounts for `TheStickBoy` and `ItzKuroYT`.

## 3. Storage

For the built-in JSON storage adapter, set:

```txt
GITHUB_TOKEN=github_pat_or_classic_token
GITHUB_REPO=YourGitHubUsername/your-iconbuilds-repo
GITHUB_BRANCH=main
GITHUB_DB_PATH=data/iconbuilds-db.json
GITHUB_DB_BACKUP_PATH=data/iconbuilds-db.backup.json
```

The token needs repository contents read/write access. The frontend never receives this token.

For a local-only test, storage writes to a temp file unless `ICONBUILDS_DB_PATH` is set.

## 4. Google Login

Create a Google OAuth web client and add:

```txt
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
```

Authorized JavaScript origins:

```txt
https://minestore.org
http://localhost:4177
```

Authorized redirect URIs:

```txt
https://minestore.org/api?action=googleCallback
http://localhost:4177/api?action=googleCallback
```

Google accounts are treated as verified when Google returns a verified email address.

## 5. Resend Email Verification

Add:

```txt
RESEND_API_KEY=re_xxxxxxxxx
RESEND_FROM_EMAIL=IconBuilds <verify@noreply.yourdomain.com>
```

Email/password users must verify by code before accessing account downloads, purchases, reviews, or admin tools.

## 6. Stripe

Add:

```txt
STRIPE_SECRET_KEY=sk_live_xxxxxxxxx
STRIPE_WEBHOOK_SECRET=whsec_xxxxxxxxx
```

Create a Stripe webhook endpoint:

```txt
https://minestore.org/api?action=stripeWebhook
```

Listen for:

```txt
checkout.session.completed
```

Prices are read from the server-side resource record. The frontend cannot grant purchases or change prices.

## 7. Protected Downloads

Admins may attach a protected file URL to a resource. Users do not see that URL directly. They receive a short-lived `/api?action=downloadFile&token=...` URL after the API confirms they own or added the resource.

For stronger production storage, place resource files in private object storage and use signed URLs.

## 8. SEO

The project includes:

```txt
robots.txt
sitemap.xml
api?action=sitemap
clean resource URLs at /resources/resource-name/
server-rendered resource pages
canonical URLs
Open Graph metadata
structured data for published resources
noindex private pages
```

Submit this sitemap in Google Search Console:

```txt
https://minestore.org/sitemap.xml
```

`/sitemap.xml` is generated by the API and includes published resources from storage.

## 9. Local Testing

Run:

```txt
npm run check
npm run smoke
npm run static
```

Local URL:

```txt
http://localhost:4177
```
